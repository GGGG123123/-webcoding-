# Vibe Coding 过程记录

## 作业目标

用 React + Vite + Three.js 实现网页端 3D 仿真车模 demo，重点呈现新能源汽车中控大屏风格、3D 可交互、车控状态联动和语音控制能力。

## 阶段划分

### 阶段 1：MVP 车模与控制台

目标：

- 搭建 React + Vite 项目。
- 使用 @react-three/fiber 和 @react-three/drei 渲染 3D 场景。
- 优先加载用户下载的本地 GLTF 车模，并保留模型原本颜色。
- 实现鼠标拖动旋转、滚轮缩放。
- 实现基础车窗和车灯控制按钮。

验收：

- `npm run dev` 可启动。
- 页面显示 3D 车模。
- 车模展示背景为白色。
- 车模能旋转和缩放查看。
- 点击按钮后，车窗或车灯状态能同步到模型。

### 阶段 2：车控交互增强

目标：

- 支持点击车窗直接控制对应车窗。
- 增加车窗开合动画。
- 丰富车控项，例如车门、后备箱、充电口、氛围灯。

验收：

- 3D 模型部件可点击。
- 控制面板与 3D 部件状态保持一致。
- 状态变化具有明确视觉反馈。

### 阶段 3：语音控车

目标：

- 使用浏览器 Web Speech API 接入语音识别。
- 支持“打开左前车窗”“关闭所有车窗”“打开车灯”等命令。
- 在页面展示识别结果和执行状态。

验收：

- 浏览器允许麦克风权限后可识别中文命令。
- 命令执行后模型和控制面板同步更新。
- 无法识别时给出友好反馈。

### 阶段 4：交付整理

目标：

- 完善 README。
- 完善 Vibe Coding 过程记录。
- 录制演示视频。
- 整理最终代码。

验收：

- 代码可运行。
- 文档可说明开发过程、功能和运行方式。
- 视频能演示 3D 查看、按钮控车和语音控车。

## 当前迭代记录

### v0.1 MVP

用户需求：

- 先做最小可运行版本。
- 网页中能显示 3D 车模。
- 支持鼠标拖动旋转。
- 页面有基础车控按钮。

实现思路：

- 使用 Vite React 模板初始化项目。
- 使用 `Canvas`、`OrbitControls`、`Environment`、`ContactShadows` 搭建白色棚拍 3D 视图。
- 使用本地 `public/models/nio-et7/scene.gltf` 作为优先展示车模。
- 保留 Three.js 几何体车模作为资源缺失时的兜底方案。
- 用 React state 管理 `windows` 和 `lightsOn`。
- 控制面板按钮调用状态更新函数，状态再传给 3D 车模。

已完成：

- 项目初始化。
- 第一版中控风格界面。
- 第一版本地 GLTF 车模展示。
- 白色展示背景。
- 车窗按钮控制。
- 车灯按钮控制。
- 点击车窗切换对应车窗状态。

下一步：

- 根据最终模型结构优化车窗开合动画。
- 接入语音控车。
- 补充录屏脚本和最终交付文档。

### v0.2 外部车模接入准备

用户希望使用 Sketchfab 上的 `2022 NIO ET7` 车模替换临时几何车，当前模型文件已放入 `public/models/nio-et7/`。

模型信息：

- 来源：Sketchfab
- 页面：https://sketchfab.com/3d-models/2022-nio-et7-f6dc8ff8c34040aba90bec8b248bcf3f
- 作者：Ddiaz Design
- 授权：CC Attribution-NonCommercial-ShareAlike
- 模型规模：约 561.4k triangles，311.5k vertices

实现思路：

- 由于 Sketchfab 下载接口需要登录认证，项目不直接匿名下载模型文件。
- 新增 `VehicleModel` 组件，优先检测并加载 `/models/nio-et7/scene.gltf`。
- 如果模型文件不存在，自动回退到当前 Three.js 几何车模。
- 在真实车模上识别可控玻璃 mesh，保留现有按钮和车窗/车灯状态联动。
- 白色背景展示，并保留车模原本车漆和贴图。

验收：

- 模型文件不存在时，页面不白屏，继续显示临时几何车。
- 用户下载并放置 `scene.gltf` 后，刷新页面可加载 NIO ET7 模型。
- 控制面板状态逻辑继续可用。
